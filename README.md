# 🎯 Minha forma de trabalhar

Ao iniciar qualquer projeto procuro responder algumas perguntas antes mesmo de escrever o primeiro código.

> Como esse sistema poderá crescer?

> Como ele poderá ser mantido daqui a alguns anos?

> Como reduzir riscos?

> Como garantir segurança?

> Como facilitar futuras evoluções?

Essas respostas normalmente definem toda a arquitetura da aplicação.

Por esse motivo, gosto de dedicar bastante tempo ao planejamento antes da implementação.

Ao longo dos projetos que desenvolvo, percebi que um software raramente falha apenas por causa do código.

Na maioria das vezes, os problemas surgem por decisões tomadas antes mesmo da primeira linha ser escrita.

Por esse motivo, gosto de dedicar tempo ao planejamento da solução, entendendo o problema, avaliando possíveis abordagens e escolhendo tecnologias que realmente façam sentido para aquele cenário.

Acredito que um bom desenvolvedor não é aquele que conhece muitas ferramentas, mas aquele que sabe escolher a ferramenta certa para cada desafio.
